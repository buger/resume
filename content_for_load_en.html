AIS Hydra, 2007-2010, Latera software

«Hydra» — automated billing system for telecommunications services for small and medium-sized operators.

Ruby RubyOnRails Oracle PL/SQL Stored Procedures Javascript Prototype Testing Selenium Cucumber

System core is written in Oracle with web-interface on RubyOnRails. It has been written a lot of Python modules too. It is really a huge project with tens of thousands lines of code. Despite its size, it can be developed and maintained only by few programmers. This was achieved through a unique system for the automatic generation of interfaces that allowed to minimize the amount of code.

Much attention was paid to the user interface. We created many new interface elements that significantly simplify the work in the system. I effectively used modern technologies such as AJAX and Comet.

I implemented integration testing system based on Cucumber and Selenium, which allows instantly find error.

At the moment the project Hydra being actively developed and sold successfully.

I grew it from scratch and very proud of it. Today it's a product with one of the best user interface on the market in it segment.

Chromus (ex. Last.fm free music player), 2010 - now

Music player for web-browser

Future Browser Extensions Chrome Firefox Safari Opera Javascript JQuery

Have you ever thought of making web-browser music player? I did. And seems that people really enjoyed it. Some statistics:

  • — About 4 million plays per month
  • — 80k+ users
  • — More than 6 months on the TOP of the best extension for Google Chrome.

This is extension that works in Google Chrome. It integrates with Last.fm site, turning it into a large music library.

Now I'm developing new version that will work in all browsers. It will have significantly improved interface and features that cause a big impact. Release is planned on early 2011.

To be honest: this project - my favorite. It fundamentally changes your relationship with music: once tasted, it is hard to abandon it. I and my users no longer use the local music library.

PointOut, 2010 - now

Collaborative discussion of the images.

Javascript JQuery HTML5 CSS3 Python Google App Engine

The project is quite young, but has big ambitions. It's goal - to create a convenient way for collaborative discussion of screenshots, wireframes of any other images. It is interesting both for business (communication with remote developers and designers, testers and tech-support) and for ordinary people (commenting on photos in social networks).

I am satisfied with my work. All the main functionality is implemented on the client, the server part is really small. Project uses modern web technologies such as File API and Google Channels API (for realtime user interaction).
